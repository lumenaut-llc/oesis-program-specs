# Release v1.5

## Purpose

Provide an explicit bridge-stage packet lane for `v1.5` materials without
implying that a full public `v1.5` release packet already exists.

## What `v1.5` means here

This lane documents the minimum bridge from parcel sensing into response
measurement.

Its center of gravity is:

`hazard -> house state -> action -> measured outcome`

not a broad public-marketing release or a mature controls platform.

## Reading order

1. `v1.5-scope.md`
2. `v1.5-product-surface.md`
3. `v1.5-acceptance-spec.md`

## Guardrail

`v1.5` is primarily a capability-stage bridge in the architecture and contracts.
If public-release materials are added here later, they should clearly explain
how that public label relates to:

- program-phase labels
- capability stages
- deployment maturity
- runtime asset lanes
