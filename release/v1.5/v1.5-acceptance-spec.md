# v1.5 acceptance specification

## Purpose

Document the minimum proof conditions for treating the `v1.5` bridge lane as
real rather than aspirational.

## Acceptance rule

The minimum bar is one honest closed-loop chain of:

`hazard -> house state -> action -> measured outcome`

The result must preserve evidence-quality limits instead of silently converting
weak support data into strong certainty.

## Minimum required artifacts

- one hazard record or hazard window
- one house-state record
- one action or intervention record
- one verification or outcome record
- enough metadata to interpret the relationship between them

## Minimum bridge objects

- `house_state`
- `intervention_event`
- `verification_outcome`

Useful additional support:

- equipment-state snapshot
- building or site metadata
- power-state posture

## First preferred proof path

Smoke protection remains the preferred first acceptance path because it offers a
bounded and measurable loop:

- outdoor smoke evidence at trigger time
- indoor PM response
- recirculation / fan / purifier posture where available
- logged action
- indoor improvement over a roughly 30 to 90 minute response window

## Explicit non-acceptance cases

Do not treat the lane as accepted if it only has:

- more outdoor weather sensing
- draft compatibility records without response verification
- recommendations without measured outcomes
- support-object schemas without a linked closed-loop example
