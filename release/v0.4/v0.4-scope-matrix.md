# v0.4 Scope Matrix

## Purpose

Map the v0.4 release surface against the v0.3 baseline.

## Sign-off sentence

**v0.4** means: mature node lifecycle, installation metadata, evidence
composition weighting, and deployment-quality flags — without claiming trust
scoring as a product surface, governance enforcement, or intervention logic.

## Software scope

| Surface | In v0.4 scope | Status | Notes |
|---------|--------------|--------|-------|
| Node registry lifecycle (register/bind/disable/replace/retire) | Yes | Partial | Schema exists; lifecycle flow not implemented |
| Installation metadata capture | Yes | Docs-only | Contract formalized in v1.0 lane; no capture surface |
| Evidence composition weighting | Yes | Planned | Inference must weight observations by source class and quality |

| Trust scoring (full product surface) | No | Deferred to v1.0 | v0.4 builds the inputs; v1.0 composes them |
| Sharing/consent enforcement | No | Deferred to v0.5 | |
| Append-only history | No | Deferred to v1.0 | |
| Equipment-state / house-state | No | Deferred to v1.5 | |

## Governance scope

| Surface | In v0.4 scope | Status | Notes |
|---------|--------------|--------|-------|
| Private-by-default (architectural) | Yes | Implemented | Carries forward |
| Consent / revocation / sharing | No | Deferred to v0.5 | |

## Explicit non-goals for v0.4

- Full trust scoring as a composed product surface (v1.0)
- Governance enforcement or sharing controls (v0.5)
- House-state, equipment-state, or intervention surfaces (v1.5)
- Weather-PM or thermal observation families
- Append-only observation history
- Shared-map or neighborhood signals
