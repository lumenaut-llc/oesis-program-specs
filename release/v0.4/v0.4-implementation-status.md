# v0.4 Implementation Status

## Purpose

Track per-surface implementation status for the v0.4 release.

## Status key

- `implemented` — code exists, acceptance test passes
- `partial` — code or contract exists but acceptance test not yet passing
- `docs-only` — specification exists, no runtime implementation
- `planned` — in scope but no specification or implementation
- `n/a` — not in v0.4 scope

## Acceptance test coverage note

The v0.4 acceptance test (`make oesis-v04-accept`) validates that
`multi_node_evidence` exists with `observation_count >= 2`, `indoor_count >= 1`,
and `outdoor_count + sheltered_count >= 1`. It validates registry metadata
binding into provenance. It does **not** assert that quality weighting changes
inference output values.
