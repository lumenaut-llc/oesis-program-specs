# v0.5 Scope Matrix

## Purpose

Map the v0.5 release surface against the v0.4 baseline.

## Sign-off sentence

**v0.5** means: governance enforced as runtime behavior — consent gates
sharing, revocation stops it, retention has owners, export produces auditable
output — without claiming full consumer governance UX, intervention logic,
or trust scoring as a composed product.

## Hardware scope

| Surface | In v0.5 scope | Status | Notes |
|---------|--------------|--------|-------|
| All v0.4 hardware | Yes | v0.4 target | Carries forward; no new hardware in this slice |

v0.5 is a software and governance slice. No new hardware is introduced.

## Governance scope

| Surface | In v0.5 scope | Status | Notes |
|---------|--------------|--------|-------|
| Private-by-default (architectural) | Yes | Implemented | Carries forward |
| Consent enforcement (runtime) | Yes | Docs-only | **Must become implemented** |
| Revocation (runtime) | Yes | Docs-only | **Must become implemented** |
| Retention cleanup (operational) | Yes | Partial | **Must have named owners and schedule** |
| Export controls (auditable) | Yes | Partial | **Must produce complete, validated output** |
| Operator access logging (operational) | Yes | Partial | **Must cover all parcel-linked admin actions** |
| Sharing settings (product surface) | Yes | Docs-only | **Must be configurable by operator** |
| Custody tiers (query-time eligibility) | Yes | Docs-only | **Must enforce custody_tier in sharing queries** |
| Mark-not-delete revocation semantics | Yes | Docs-only | **Must implement mark-not-delete** |

## Explicit non-goals for v0.5

- Full consumer governance UX with rich settings panels
- Full audit trail product with timeline views
- Trust scoring as a composed product surface
- Shared-map or neighborhood signal governance
- House-state or intervention governance (v1.5 surfaces)
- Automated retention enforcement (manual-with-tooling is acceptable)
