# v0.5 Gap Register

## Purpose

Classify gaps between the current implementation state and v0.5 readiness.

## Gap classification key

- **Blocker** — must resolve before claiming v0.5
- **Defer** — acceptable to track; schedule for later slice
- **Carry-forward** — inherited from earlier registers, not v0.5 scope

## Gap register

| ID | Gap | Severity | Owner area | Current status | Next action |
|----|-----|----------|-----------|----------------|-------------|
| V05-G1 | Consent enforcement: sharing path checks for valid, unrevoked consent before emitting data | **Blocker** | Technical / governance | Docs-only — consent-store schema exists; no query-time check | Implement consent check: query consent-store for matching scope, data_classes, and revoked_at == null before sharing |
| V05-G2 | Revocation behavior: revoked_at set on consent record reliably stops future sharing | **Blocker** | Technical / governance | Docs-only — mark-not-delete semantics documented; no enforcement | Implement revocation: set revoked_at → subsequent sharing queries return empty for that scope |
