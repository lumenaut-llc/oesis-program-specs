# v0.1 gap register

## Purpose

Classify gaps uncovered during the v0.1 completeness review (2026-04-08) as **v0.1 blocker**, **deferred (v0.2+ / later pre-1.0)**, or **PRD / product-only** (not required for the narrow reference slice). Assign a default owner where the program already names one.

## Severity key

- **Blocker** — must resolve before calling the v0.1 reference slice “complete” for a stated audience.
- **Defer** — acceptable to ship/track as v0.1 reference; schedule for a later slice or product phase.
- **PRD-only** — required for full “current v1” product, not for v0.1 reference.

| ID | Gap | Severity | Owner (default) | Next artifact |
| --- | --- | --- | --- | --- |
| G1 | Named BOM vendors / SKU-level procurement (matrix already flags) | Defer for lab v0.1; **Blocker** for naive third-party kitting | Release / hardware | `integrated-parcel-kit-bom.md` + checklist: add preferred suppliers or “bring your own” rules |

| G3 | Continuous Wi-Fi (or other) transport from node to ingest with TLS, identity, retries | Defer | Technical | Spec transport lane; v0.1 assumes serial extract or HTTP demo with manual packet |
