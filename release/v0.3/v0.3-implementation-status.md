# v0.3 Implementation Status

## Purpose

Track per-surface implementation status for the v0.3 release.

## Status key

- `implemented` — code exists, acceptance test passes
- `partial` — code or contract exists but acceptance test not yet passing
- `docs-only` — specification exists, no runtime implementation
- `planned` — in scope but no specification or implementation
- `n/a` — not in v0.3 scope

## Acceptance test coverage note

The v0.3 acceptance test (`make oesis-v03-accept`) validates that flood packets
normalize correctly (observation_type, water_depth_cm, distance_cm,
dry_reference_distance_cm present) and that flood-node-01 is in the parcel
context. It does **not** assert that flood observations change parcel-state
hazard probabilities or shelter status.
