# v0.2 Gap Register

## Purpose

Classify gaps between the current implementation state and v0.2 readiness.

## Gap classification key

- **Blocker** — must resolve before claiming v0.2
- **Defer** — acceptable to track as known gap; schedule for later slice
- **Carry-forward** — inherited from v0.1 gap register, not v0.2 scope

## Gap register

| ID | Gap | Severity | Owner area | Current status | Next action |
|----|-----|----------|-----------|----------------|-------------|

| V02-G2 | Node registry lifecycle: binding two nodes to one parcel with enable/disable semantics | **Blocker** | Technical | Partial — schema exists; lifecycle not exercised in runtime | Implement register/bind/disable flow for two-node parcel |
