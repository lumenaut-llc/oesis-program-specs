# v1.0 launch checklist

## Purpose

Define comprehensive launch gates for the v1.0 release, organized by tier:

- **Tier A: Internal reference v1.0** -- engineers and internal reviewers validate the widened parcel-kit slice; no external participant data.
- **Tier B: External pilot** -- any real household, participant, or identifiable parcel data; stop-ship rules apply.

Each gate includes the tier it applies to, current status, and the evidence needed to close it. Gates marked "both" must pass for either tier. Gates marked "Tier B" are additionally required before any external participant data enters the system.

## Status key

- `not started` -- no work has begun
- `in progress` -- work is underway but not yet verifiable
- `complete` -- evidence exists and has been reviewed

## Stop-ship triggers

Any of the following conditions should halt progression to the stated tier:

### Tier A stop-ship

- `make oesis-validate` or `make oesis-v10-validate` fails
- Reference pipeline cannot produce a parcel view from bench-air input
- Public copy implies product behavior that the reference stack cannot demonstrate

### Tier B stop-ship

- Any Tier A stop-ship condition
- Sharing enabled without matching consent notice
- Revocation does not reliably stop future sharing
- Exact parcel-linked contributed data published without explicit designation
- Participant-facing materials contain directive safety claims
- No named incident contacts for pilot operations
- Ingest accepts live node data without identity verification
