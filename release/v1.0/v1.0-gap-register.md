# v1.0 gap register

## Purpose

Classify gaps between the current implementation state and v1.0 readiness, so the program can distinguish blockers from deferred work and avoid treating documentation as product behavior.

This register builds on the v0.1 gap register (`../v.0.1/v0.1-gap-register.md`) and reflects the wider scope described in `program/operating-packet/09-phasing-v0.1-v1.0-v1.5.md` (v1.0 section) and `v1.0-scope-matrix.md`.

## Gap classification key

- **Blocker** -- must resolve before claiming v1.0 readiness for the stated tier (Tier A internal reference or Tier B external pilot).
- **Defer** -- acceptable to track as known gap in v1.0; schedule for a later slice or capability stage.
- **PRD-only** -- required for full consumer product experience, not for v1.0 as defined in the phasing doc.
