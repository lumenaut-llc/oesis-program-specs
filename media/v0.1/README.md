# Media

Diagrams, renders, photos, screenshots, and visual walkthrough assets.

## Lane contract

- **Baseline lane**: `v0.1/` media assets are the accepted default visuals for the
  current documentation and release lanes.
- **Additive lanes**: add explicit version folders (for example `v1.0/`,
  `v1.5/`) only when visuals materially diverge by lane.
- **Compatibility policy**: when media paths move, keep index pointers or
  redirect notes so high-traffic links remain resolvable.
- **Canonical mapping**: visual claims should remain consistent with
  `../../architecture/system/version-and-promotion-matrix.md` and the matching
  lane packet in `../../release/`.

## Mutation rule

Do not silently replace baseline visuals with future-lane concepts in-place.
Add lane-specific assets when meaning diverges.
