# Internal Operator Access Policy

## Purpose

Define the minimum internal access expectations for maintainers, operators, support, and researchers who may interact with parcel-linked data.

## Governing rule

Access to parcel-linked private data should be need-based, time-bounded where practical, and logged.

## Minimum access rules

- private parcel data should not be browsed casually
- operators should use the least detailed access that meets the operational need
- support access to parcel-linked data should be logged
- engineering debug access should prefer synthetic or example data first
- research access should follow the separate pilot or research approval path

## Approval triggers

Extra review should be required for:

- bulk access to parcel-linked records
- access for purposes beyond product operation, security, or approved support
- exports of parcel-linked data
- access to data involved in a complaint, incident, or contested output

## Logging requirements

Log at minimum:

- who accessed the data
- when access occurred
- what class of data was accessed
- why access was requested
- whether any export occurred

## Prohibited practices

- informal copies of parcel-linked data on personal machines
- using production household data as routine demo material
- using parcel-linked data for unrelated analytics, marketing, or profiling
- bypassing revocation or deletion outcomes through retained side copies
