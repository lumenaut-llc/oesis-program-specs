## Core stance

The target-lane software story should not read as a simple widening of outdoor
weather hardware support.

It should read as the software path needed to strengthen a broader fielded
parcel-intelligence kit:

ingest quality -> parcel binding -> evidence confidence/freshness -> clearer
parcel view semantics

## Scope of this lane

These docs describe:

- the stronger quickstart story
- stronger parcel-kit software inputs and records
- the distinction between `v1.5` bridge surfaces and `v2.5` compatibility work
- software treatment of non-node support surfaces as first-class records

The closed-loop bridge narrative (`hazard -> house state -> action -> measured
outcome`) belongs in `../v1.5/`.
